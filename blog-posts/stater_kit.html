The SOC Starter Kit - Tools & Frameworks

Published: 2025-10-25 · Written by twh4t

The best defense starts with visibility. This guide provides a structured starter kit to help aspiring analysts build a solid foundation using industry-standard tools and frameworks.

1 - MITRE ATT&CK: The SOC Threat Map

MITRE ATT&CK is the de facto language for adversary behavior. Use it to map detections to attacker tactics and techniques, validate coverage, and communicate impact across teams.

How to use it:
  • Map existing detections to ATT&CK techniques and identify gaps.
  • Use ATT&CK Navigator to visualize coverage by tactic.
  • Run simulations with Atomic Red Team or Caldera to validate detections.

2 - Threat Intelligence: Abuse.ch · OTX · MISP · VirusTotal

Threat intelligence enriches alerts with external context. Automate lookups so every IOC in your SIEM carries context and a score.

Feeds & platforms:
  • Abuse.ch - botnet & malware C2 tracking (Zeus, Emotet, etc.).
  • AlienVault OTX - community-driven shared IOCs.
  • MISP - share structured intelligence with teams and partners.
  • VirusTotal - file/hash/domain reputation & multi-vendor scan results.

Example enrichment flow: SIEM alert → extract IOC → query OTX & VirusTotal → add score + tags

3 - Network Visibility: Snort, Suricata, Zeek & Wireshark

Network telemetry gives you metadata and payload insight - priceless when endpoints lie or are offline.

IDS / NIDS

  • Snort - classic signature-driven detection, lightweight and rule-focused.
  • Suricata - multithreaded, supports file extraction, TLS fingerprinting, and EVE JSON output for SIEM ingestion.
  • Zeek (Bro) - rich network metadata generation (DNS logs, HTTP, SSL, conn logs) - best used for hunting and context.

Packet analysis - Wireshark basics

Wireshark is your packet-level microscope. Key skills:

  • Follow TCP streams and inspect payloads.
  • Identify DNS anomalies and suspicious long-lived TCP/HTTP beaconing.
  • Understand the TCP 3-way handshake, retransmits, and abnormal flags.
    • # quick tcpdump capture example
sudo tcpdump -i eth0 -w capture.pcap
# then open in Wireshark for analysis

4 - Endpoint Telemetry: Windows Event Logs + Sysmon + Osquery

Strong endpoint telemetry is the backbone of detection. Native Windows logs are good - Sysmon and osquery make them great.

Windows Event Logs

  • Event ID 4624 / 4625 - account logon success / failure (watch impossible travel).
  • Event ID 4688 - process creation (track suspicious child processes).
  • Event ID 7045 - new service installed (persistence indicator).

Sysmon

Sysmon extends logging with process hashes, parent-child relationships, network connections, and file creation events - indispensable for forensic timelines.

osquery

Use osquery to query live endpoint state with SQL-like queries. Examples:

-- find running PowerShell processes
SELECT pid, name, cmdline FROM processes WHERE name = 'powershell.exe';

5 - SIEM & Monitoring: Splunk · Wazuh · Security Onion

Your SIEM is where telemetry becomes insight. Pick a platform that fits scale and budget, then tune rules and parsers.

  • Splunk - enterprise-grade search, dashboards, and alerting.
  • Wazuh - open-source SIEM + XDR with good agent coverage.
  • Security Onion - an integrated distro bundling Zeek, Suricata, Wazuh, and Kibana for visibility and hunting.

6 - Forensics: Autopsy · Volatility · FTK Imager

When an incident escalates, you need disk and memory analysis to find stealthy persistence and in-memory artifacts.

  • Autopsy - disk forensics GUI for timelines, deleted file recovery, and web history.
  • Volatility - memory forensics; enumerate processes, handles, DLLs, injected threads, and more.
  • FTK Imager - create forensically-sound disk images for offline analysis.

7 - Cloud Visibility: AWS · Azure · GCP

Cloud is part of the attack surface. Instrument it with native telemetry and treat cloud logs like first-class evidence.

  • AWS GuardDuty - threat detection using CloudTrail, VPC flow logs, and DNS logs.
  • Azure Defender & Sentinel - cloud-native detection + SIEM/SOAR.
  • GCP Security Command Center - asset risk visibility and event monitoring.

Monitor:

  • IAM changes and new role creations
  • Unexpected API calls or console logins
  • Unusual outbound connections from cloud workloads

8 - Putting It Together: Workflow & Automation

Visibility + intel + automation = scalable SOC operations.

  1. Detect: SIEM alerts, IDS events, EDR telemetry.
  2. Enrich: Auto-query OTX / VirusTotal / internal intel.
  3. Investigate: Pull Sysmon, Zeek logs, packet captures, and run osquery queries.
  4. Contain: Isolate host via EDR, block IPs on firewall, revoke cloud keys if needed.
  5. Document: Timeline, root cause, remediation, and lessons learned.
# example: simple Python pseudo-flow for IOC enrichment
ioc = "1.2.3.4"
vt = query_virustotal(ioc)
otx = query_otx(ioc)
if vt.malicious or otx.seen:
    tag_alert(alert_id, "threat-intel:high")

9 - Analyst Mindset & Final Tips

Tools change - mindset doesn't. Focus on:

  • Learn your environment before chasing alerts.
  • Automate noise so analysts hunt patterns, not spam.
  • Document everything - clear notes accelerate future response.
  • Map detections to ATT&CK - this reveals gaps and drives measurable improvements.

10 - Resources & Next Steps

Start building a lab that combines these components:

  • Deploy Security Onion for Zeek + Suricata + Wazuh
  • Ship Sysmon logs to a SIEM (Splunk or Wazuh)
  • Integrate VirusTotal & OTX lookups via simple scripts or a SOAR playbook
  • Practice detection validation with Atomic Red Team
